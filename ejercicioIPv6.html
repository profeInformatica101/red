Ejercicio 2 - IPv6

Definición del problema:

Indique cuál de las siguientes expresiones de dirección IPv6 es incorrecta, y explique por qué es incorrecta:

  • a) 2001:0DB8:F181:0000:0000:0000:0000:0000
  • b) 2001:0DB8:F181:0:0:0:0:0
  • c) 2001:DB8:F181:0:0:0:0:0
  • d) 2001:DB8:F181::0:0:0:0
  • e) 2001:DB8:F181::0:0:0
  • f) 2001:DB8:F181::0:0
  • g) 2001:DB8:F181::

Respuestas y explicaciones:

Indique cuál de las siguientes expresiones de dirección IPv6 es incorrecta, explique el por qué:

  • a) 2001:0DB8:F181:0000:0000:0000:0000:0000 (Correcta)

    Esta es una dirección completa de 128 bits, con todos los grupos de 16 bits presentes, por lo tanto es válida.

  • b) 2001:0DB8:F181:0:0:0:0:0 (Correcta)

    Esta dirección está abreviada correctamente utilizando ceros, y no se usa `::` para compactar. También es válida.

  • c) 2001:DB8:F181:0:0:0:0:0 (Correcta)

    Esta es similar a la opción b), con una abreviación válida de ceros en el formato correcto.

  • d) 2001:DB8:F181::0:0:0:0 (Correcta)

    El uso de `::` está correctamente empleado para representar múltiples ceros consecutivos. Es válida.

  • e) 2001:DB8:F181::0:0:0 (Correcta)

    Similar a la opción d), el `::` se utiliza adecuadamente para compactar los ceros consecutivos. Es válida.

  • f) 2001:DB8:F181::0:0 (Correcta)

    Esta dirección usa correctamente el `::` para abreviar los ceros consecutivos y es válida.

  • g) 2001:DB8:F181:: (Incorrecta)

    Esta es incorrecta. Aunque se usa `::` para abreviar ceros consecutivos, faltan al menos 16 bits (un grupo de 16 bits) para que la dirección tenga 128 bits en total, por lo que no es válida.

Explicación General:

La opción g) es incorrecta porque no tiene el número completo de bloques que se necesitan para formar una dirección IPv6 válida. Al usar ::, se deben rellenar los ceros restantes, pero aún así la dirección debe tener 128 bits en total. En esta dirección faltan al menos 16 bits (un bloque) para que sea válida.

Ejercicio: Asignación de Direcciones IPv6 y Subnetting

Enunciado:

Dado el siguiente esquema de red, asigna direcciones de red e IPs para el Router 1 (R1) en sus distintas interfaces, realizando subnetting con direcciones IPv6. Tomaremos como base el prefijo de enrutamiento global 2001:0DB8:ACAD::/48.

Paso 1: ¿Qué es IPv6 y por qué es diferente a IPv4?

IPv6 es la versión más reciente del protocolo de Internet (IP) y está diseñado para reemplazar IPv4 debido a la limitación en la cantidad de direcciones disponibles en IPv4. IPv6 usa direcciones de 128 bits, lo que proporciona un espacio de direcciones mucho mayor comparado con los 32 bits de IPv4. Las direcciones IPv6 se representan en hexadecimal y están divididas en 8 grupos de 16 bits, separados por dos puntos :. Ejemplo: 2001:0DB8:ACAD::.

Paso 2: ¿Qué es el subnetting en IPv6?

El subnetting es el proceso de dividir una red en subredes más pequeñas. En este caso, se nos proporciona una red principal con el prefijo 2001:0DB8:ACAD::/48, lo que significa que los primeros 48 bits están reservados para la red global. Con 128 bits totales en una dirección IPv6, nos quedan 80 bits para dividir entre subredes e identificadores de hosts.

Paso 3: ¿Cuántas subredes se pueden crear?

Con 16 bits disponibles para hacer subnetting (es decir, dividir la red en partes más pequeñas), podemos crear hasta 65.536 subredes diferentes (216 = 65.536).

Paso 6: ¿Por qué se utilizan subredes?

  • Organización y segmentación: El uso de subredes permite dividir grandes redes en segmentos más pequeños, lo que facilita la administración y reduce el tráfico innecesario.
  • Seguridad: Al segmentar la red, se puede aplicar más fácilmente control de acceso y medidas de seguridad por subred.
  • Escalabilidad: IPv6 ofrece un espacio de direcciones mucho mayor que IPv4, lo que permite crear miles de subredes sin necesidad de re-diseñar la red.

Ejercicio 10

Pregunta: Convertir la dirección 192.168.20.112 en una dirección mapeada IPv6.

Solución:

Para convertir una dirección IPv4 en una dirección IPv6 mapeada, seguimos los siguientes pasos:

  1. Convertir cada octeto de la dirección IPv4 192.168.20.112 en su forma hexadecimal:
    • 192 → C0
    • 168 → A8
    • 20 → 14
    • 112 → 70
  2. Combinar los valores hexadecimales: C0A8:1470
  3. Añadir el prefijo ::ffff: para formar la dirección IPv6 mapeada:

Dirección IPv6 mapeada: ::ffff:c0a8:1470

Ejercicio 11

Pregunta: ¿Qué dirección es asignada por defecto cuando no tenemos un servidor DHCP en IPv6, suponiendo que la dirección MAC del equipo es 52:74:f2:b1:a8:7f?

Solución:

Cuando no se utiliza un servidor DHCP en IPv6, se genera una dirección link-local utilizando la dirección MAC del equipo, siguiendo la convención EUI-64. A continuación, los pasos para obtener la dirección:

  1. Dividir la dirección MAC en dos partes de 24 bits:
    • MAC: 52:74:f2:b1:a8:7f
    • Primera parte: 52:74:f2
    • Segunda parte: b1:a8:7f
  2. Insertar ff:fe en el medio:
    • Resultado: 52:74:f2:ff:fe:b1:a8:7f
  3. Invertir el bit 7 (bit U/L):
    • La primera parte 52 en binario es 01010010.
    • Al invertir el bit 7, el valor cambia a 01010000, lo que convierte 52 en 50.
    • Nuevo valor: 50:74:f2:ff:fe:b1:a8:7f.
  4. Añadir el prefijo fe80::/64 para obtener la dirección link-local completa:
    • Dirección IPv6 generada: fe80::5074:f2ff:feb1:a87f/64

Respuesta final: La dirección IPv6 que se asigna por defecto es fe80::5074:f2ff:feb1:a87f/64.

Ejercicio 9

Pregunta: ¿Cuál de las siguientes direcciones es la abreviatura más corta para la dirección IP 3FFE:1044:0000:0000:00AB:0000:0000:0057?

Opciones:

  • a) 3FFE:1044::AB::57
  • b) 3FFE:1044::00AB:0057
  • c) 3FFE:1044::AB::57
  • d) 3FFE:1044::AB:0:57

Solución:

Para abreviar una dirección IPv6, debemos seguir una serie de reglas que nos permiten simplificar la escritura de las direcciones sin perder información. La dirección original que se nos da es:

3FFE:1044:0000:0000:00AB:0000:0000:0057

Paso 1: Omitir ceros a la izquierda

En IPv6, cualquier grupo de ceros a la izquierda de un segmento de la dirección puede omitirse. Aplicamos esto a los siguientes segmentos:

  • 00AB se convierte en AB
  • 0057 se convierte en 57

Por lo tanto, la dirección se convierte en:

3FFE:1044:0000:0000:AB:0000:0000:57

Paso 2: Usar :: para agrupar ceros consecutivos

En IPv6, podemos usar :: para reemplazar un grupo consecutivo de segmentos que contienen solo ceros. Este mecanismo solo puede ser utilizado una vez en la dirección. En este caso, tenemos dos grupos consecutivos de ceros:

  • 0000:0000 aparece dos veces seguidas, y podemos reemplazarlos con ::.

Después de aplicar la abreviatura, obtenemos la siguiente dirección:

3FFE:1044::AB:0000:57

Paso 3: Revisar si podemos aplicar más abreviaciones

Ya hemos utilizado :: para los ceros consecutivos. Sin embargo, aún tenemos un grupo de ceros adicionales entre AB y 57:

  • El segmento 0000 que aparece antes de 57 puede ser escrito como 0.

Entonces, la dirección completa y abreviada sería:

3FFE:1044::AB:0:57

Paso 4: Verificar las opciones dadas

Ahora que tenemos la abreviatura correcta, comparamos con las opciones propuestas:

  • a) 3FFE:1044::AB::57 → Incorrecto. No se pueden usar dos :: en una dirección IPv6.
  • b) 3FFE:1044::00AB:0057 → Incorrecto. 00AB y 0057 no han sido abreviados correctamente.
  • c) 3FFE:1044::AB::57 → Incorrecto. Nuevamente, se han utilizado dos ::, lo cual no es permitido.
  • d) 3FFE:1044::AB:0:57 → Correcto. Esta es la abreviatura más corta y válida.

Conclusión:

La abreviatura más corta para la dirección IP 3FFE:1044:0000:0000:00AB:0000:0000:0057 es la opción d) 3FFE:1044::AB:0:57. Esto se debe a que:

  1. Omitimos los ceros a la izquierda en 00AB y 0057.
  2. Reemplazamos los grupos consecutivos de ceros 0000:0000 con ::.
  3. No se pueden utilizar dos :: en la misma dirección.
  4. Se puede abreviar un segmento de 0000 a simplemente 0.
